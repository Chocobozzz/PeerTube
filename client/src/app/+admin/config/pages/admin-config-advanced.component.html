AUTHENTICATION

Extra auth configuration behaviors

When enabled, a user who registered with one authentication source is now allowed to login to their account using any other authentication source available.

⚠️ Only enable this if you know what you are doing, this can lead to account takeover if used in the wrong context. To be used safely, you MUST ensure that the union of each authentication source associates a globally unique uid to each unique user. (If a user is present in more than one source, it must have the same uid in all sources. If a user is not present in all sources, another user cannot take the uid of the former in the sources where it does not exist.)
